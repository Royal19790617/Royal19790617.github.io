JAVA 8 之Optional

JAVA 8 Optional是什么

  • Optional 类是一个可以为null的容器对象。如果值存在则isPresent()方法会返回true，调用get()方法会返回该对象。
  • Optional 是个容器：它可以保存类型T的值，或者仅仅保存null。Optional提供很多有用的方法，这样我们就不用显式进行空值检测。
  • Optional 类的引入很好的解决空指针异常。

JAVA 8之LAMBDA表达式

什么是LAMBDA表达式————就像sql语法加入了java代码里

关键语法是stream，主要是介绍新stream常用的方式

stream() 为集合创建串行流。

parallelStream() 为集合创建并行流。

当把一个数据结构包装成Stream后，就要开始对里面的元素进行各类操作了。常见的操作可以归类如下。

Intermediate：中间操作

  • map (mapToInt, flatMap 等)、 filter、 distinct、 sorted、 peek、 limit、 skip、 parallel、 sequential、 unordered

map

  • map/flatMap映射 把 Stream中 的每一个元素，映射成另外一个元素。

filter

  • 过滤集合中的空元素，映射保存为另一个集合

distinct

  • 对集合去重

sorted

  • 对集合排序

groupingBy

collect

joining

Terminal： 最终操作

  • forEach、 forEachOrdered、 toArray、 reduce、 collect、 min、 max、 count、 anyMatch、 allMatch、 noneMatch、 findFirst、 findAny、 iterator

forEach

Short-circuiting： 短路操作

  • anyMatch、 allMatch、 noneMatch、 findFirst、 findAny、 limit

统计

list转map

map转list

总结

  • 实际使用时，要灵活地组合使用各种语法，还有我初开始使用stream的时候觉得很好用，语法简练，不论什么情况都会想着去使用lambda语法，首先要说，孰能生巧，不多写不可能灵活运用，lambda绝对能提升代码效率，以前很长一段代码通过lambda语法很短一段就能搞定了，但是也不是说无脑用，之前有段时间感觉用lambda写出来的代码逼格很高，要写的好还是需要多看源码，知道原理，其实并不是所有代码都需要用stream，譬如一个简单的循环，for的效率是高于用stream的，原因是它内部就是用for写的，你包装了一层消耗当然要更多咯。
